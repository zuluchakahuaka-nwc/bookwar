# BOOKWAR

**BOOKWAR** — это ролевая игра с карточными боями на основе русского алфавита в сеттинге мрачного средневековья. Буквы — это одновременно оружие (гласные = атака), защита (согласные = щит), модификаторы (Ъ, Ь) и валюта. Знаки препинания дают особые способности.

Когда-то в мире звучала речь, пока древний **Хранитель Запрета** не разбросал буквы по тёмным землям и не наложил проклятье молчания. Вы — герой, который отправляется вернуть все 33 буквы алфавита.

- **Движок:** Godot 4.6 (GDScript)
- **Платформы:** Web (HTML5), Windows/Mac/Linux, Android
- **Режимы:** одиночная игра + мультиплеер (кооператив/PvP по WebSocket)

---

## 33 уровня с прогрессией сложности

Игра — это цепочка из **33 уровней**, каждый сложнее предыдущего. Сложность растёт плавно от 1 до 33: больше врагов, опаснее существа, темнее освещение, сильнее и дороже буквы.

| Уровни | Что происходит |
|--------|----------------|
| **1** — Светлая Долина | Обучение. Только знаки вопроса `?` и базовые восклицания `!`. Первые буквы А, О, М. |
| **2–3** — Лес Двубуквия, Дремучие Дубы | Появляются лесные твари с буквами, тёмные волки, тени. |
| **4–22** | Болота, пещеры, катакомбы, руины, цитадели. Враги используют всё больше букв; освещение темнеет с каждым уровнем. |
| **23–28** | Начинаются **сильные и дорогие буквы** (Ъ, Ы, Ь, Э, Ю, Я — самые мощные по инвертированному балансу). Элитные отряды. |
| **32** — Трон Хранителя Запрета | Предфинал: помощники злого волшебника **Знак** и **Звук** + элитная гвардия. |
| **33** — Колодец Букв | **ФИНАЛ: массовая битва «толпа на толпу»** — орда `?` против армии `!`, помощники волшебника (Знак, Звук) и сам босс **Хранитель Запрета** (злой волшебник, 500 HP, владеет полным алфавитом). |

Полный список уровней — в [`data/levels.json`](data/levels.json).

## Буквенная система (ядро)

- **33 буквы** — единственный ресурс: они же оружие, защита и валюта.
- **Уровень буквы** = количество собранных копий (перманентный; бой не снижает его).
- **Инвертированный баланс:** сила буквы = её позиция в алфавите. `А` (1) — слабая, `Я` (33) — самая сильная. Медленные/мощные против быстрых/слабых.
- **Заклинания** — комбинации букв (БАМ, ЩИТ, БОГ, ЖАР, …). Рецепты открываются за буквицы-валюту. См. [`data/spells.json`](data/spells.json).
- **Знаки препинания:** `...` — разговор с монстрами (дипломатия), `,` — замедление, `—` — пробой щита, и др. См. [`data/punctuation.json`](data/punctuation.json).

## Мультиплеер

Клиент на WebSocket: синхронизация позиций, чат, обмен буквами, приглашения на бой. Web-сборка ходит через тот же origin (`/ws`), нативные клиенты — напрямую.

- **Клиент:** [`scripts/multiplayer/network_manager.gd`](scripts/multiplayer/network_manager.gd)
- **Сервер-релей (Node.js + ws):** [`multiplayer/server.js`](multiplayer/server.js)
- По умолчанию клиент подключается к `ws://localhost:4567`. Свой сервер можно указать через URL-параметр `?mp=ws://host:port`.

Запуск сервера:
```bash
node multiplayer/server.js   # слушает 0.0.0.0:4567
```

## Как запустить

### Из исходников (редактор Godot)
1. Установить [Godot 4.6](https://godotengine.org/).
2. Открыть папку проекта в Godot (`project.godot`).
3. Нажать F5 (запуск `main_menu.tscn`).

### HTML5-сборка
1. В Godot: **Project → Export → HTML5** → Export.
2. Раздать папку экспорта любым статическим сервером:
   ```bash
   npx http-server builds/html5 -p 3000
   ```
3. Открыть `http://localhost:3000/`.

### Android
1. Установить Android build template (Project → Install Android Build Template).
2. Настроить keystore в **Editor → Editor Settings → Export**.
3. Export → Android.

## Управление

- **WASD / стрелки** — движение (или виртуальный D-pad на мобильных).
- **E / T** — взаимодействие / диалог с ближайшим монстром (нужно `...`).
- **F** — атаковать монстра (инициировать бой).
- **I** — инвентарь (крафт букв).
- **Q** — журнал квестов.
- **Space** — пауза.
- На мобильных все основные действия дублированы тач-кнопками HUD.

## Тестирование

E2E-тесты прогоняют реальную HTML5-сборку в headless Chromium через Puppeteer (как живой игрок): загрузка, движение, бой, инвентарь, прогрессия уровней, мультиплеер.

```bash
cd tests/e2e
npm install
npx http-server ../../builds/html5 -p 3000 &   # отдельным процессом
npx jest                                        # все тесты
npx jest component/regression_levels.test.js    # проверка 33 уровней
```
