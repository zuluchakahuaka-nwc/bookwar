## 🌐 ЛОКАЛИЗАЦИЯ ДЛЯ ВСЕХ ЯЗЫКОВ + ШРИФТЫ (масштабная задача)

> **ГЛАВНЫЙ ПРИНЦИП:** игра завязана на алфавит языка. У русского 33 буквы =
> 33 уровня, 33 «оружия/брони» (гласные=атака, согласные=щит). У других языков
> число букв ДРУГОЕ → **меняется число уровней, число предметов и баланс**.
> Локализация = НЕ перевод строк, а ПЕРЕСТРОЕНИЕ всей игры под алфавит локали.
> С китайским (тысячи иероглифов, нет фиксированного алфавита) это кандидат на
> «самую долгую игру в мире» ахахахха.

| Локаль | Букв | Уровней | Оружие/броня | Примечание |
|--------|------|---------|--------------|------------|
| ru | 33 | 33 | 33 | текущая версия |
| en | 26 | 26 | 26 | латиница |
| es | 27 | 27 | 27 | +ñ |
| fr | 26 | 26 | 26 | латиница |
| de | 30 | 30 | 30 | +ä,ö,ü,ß |
| pt | 26 | 26 | 26 | латиница |
| ar | 28 | 28 | 28 | арабский, RTL |
| zh | ??? | ??? | ??? | нет алфавита — отдельный дизайн (см. I18N-6) |

### I18N-0. Архитектура: алфавит-зависимая механика (РЕФАКТОР — основа всего)
- Сейчас `EXPECTED_LETTER_COUNT=33`, `MAP_CHAIN` (33), `levels.json` (33) — **захардкожены под русский**. Всё это должно стать **динамическим, производным от загруженного алфавита локали**.
- `data/letters_{locale}.json` — алфавит для каждого языка: {char, position, type(vowel/consonant/sign), base_power}. Универсальная формула `base_power = position` (инверсия), `speed = position` остаётся, но MAX = длине алфавита.
- На старте: `I18n.get_locale()` → грузит `letters_{locale}.json` → от него **генерируется** число уровней (= N букв), прогрессия (level k ≈ середина при k=N/2), спавнер, список букв-предметов.
- `MAP_CHAIN` / `levels.json` → генерируются процедурно из N (имена биомов — параметризовать или переиспользовать набор из N≤max).
- Баланс: число гласных/согласных РАЗНОЕ в каждом языке → разное соотношение атакующих/защитных букв → ребаланс множителей (VOWEL/CONSONANT_MULTIPLIER) per-locale.

### I18N-1. Базовая локализация (инфраструктура есть)
- Автозагрузка `I18n` (`scripts/core/i18n.gd`) + цепочка шрифтов per-locale уже работают.
- Файлы переводов: `data/i18n/{ru,en,es,fr,de,pt,ar,zh}.json` (8 локалей) — завести ключи для ВСЕХ строк UI/HUD/диалогов/легенды/мануала.
- Часть строк захардкодена по-русски (HUD, диалоги, легенда) — вынести в ключи `t("...")`.

### I18N-2. Шрифты под каждый алфавит
- Кириллица (ru) — базовый шрифт ✓
- Латынь (en/es/fr/de/pt) ✓
- **Арабский (ar)** — RTL-направление + NotoSansArabic (есть `assets/fonts/NotoSansArabic-Regular.ttf`). Проверить рендер и mirroring UI.
- **Китайский (zh)** — `NotoSansSC-Regular.otf` (8 МБ) покрывает Simplified Chinese. Для полного охвата — рассмотреть NotoSansCJK (полнее, но тяжелее).
- Проверить: `Engine.getMissingFeatures()` пуст, все глифы рендерятся (Puppeteer + Vision на ключевых экранах).

### I18N-3. Китайский — полный охват текстов (самое долгое)
- Перевести ВСЕ тексты (легенда 7 сцен, мануал 6 вкладок, диалоги всех монстров, заклинания, HUD, квесты) на zh.

- Числовые/формульные строки (`I18n.t_fmt(...)`) — проверить порядок плейсхолдеров для zh.

### I18N-4. Региональная адаптация контента
- 33 уровня (для ru): имена регионов (`region.*` в i18n) — перевести. Для остальных локалей N уровней (см. I18N-0) — имена биомов либо перевести шаблон, либо сгенерить.
- Лор/диалоги: перевести реплики Хранителя Запрета, Знака, Звука, Длинноязыкого и др.

### I18N-5. Перебаланс боя per-locale
- Разное число гласных/согласных → разный average-damage / average-shield.
- Например en: 5 гласных (AEIOU) vs 21 согласных → дефолт-defense-heavy. Ru: 10 гласных vs 21 согласных + 2 знака. Подобрать `VOWEL_MULTIPLIER`/`CONSONANT_MULTIPLIER` на локаль, чтобы бой оставался тактичным.
- Спеллы (слова из букв) — для не-русских локалей СВОИ слова-заклинания (из букв этого алфавита).

### I18N-6. Китайский — спец-дизайн (нет алфавита)
- У китайского нет конечного набора букв → модель «1 иероглиф = 1 уровень/предмет» ломается (тысячи иероглифов).
- Варианты на выбор:
  - (a) Пиньинь (26 латинских букв) — как en.
  - (b) Набор из N ключевых иероглифов (например 33 или 100 базовых) = «алфавит» для zh.
  - (c) Радикалы (214 ключей Канси) — 214 «букв» → 214 уровней (тогда zh = самая длинная версия игры).
- Решить и зафиксировать в `data/letters_zh.json`.

**Критерий готовности:** во всех 8 локалях — уровни = длине алфавита, предметы = буквам, баланс откалиброван, 0 missing-glyph на всех экранах, Puppeteer+Vision подтверждает читаемость (особенно zh/ar).

## 🔤 КРАФТ БУКВ (в инвентаре)

### C1. Механика трансформации букв
Игрок может изменить одну букву в другую, добавив модификатор:
| Из | + модификатор | → В | шанс успеха |
|----|---------------|-----|-------------|
| Б | + Ь или ' | → П | 70% |
| П | + Ь или ' | → Б | 70% |
| Ф | + Ь | → В | 70% |
| В | + Ь | → Ф | 70% |
| М | + Ь | → Н | 70% |
| Н | + Ь | → М | 70% |
| Щ | + ' | → Ш | 70% |
| Ш | + ' | → Щ | 70% |
| С | + Ь | → З | 60% |
| Д | + Ь | → Т | 60% |

**При неудаче**: буква-донор (модификатор Ь/') ТЕРЯЕТСЯ без получения новой буквы.
**При удаче**: донор расходуется, целевая буква добавляется (или +1 уровень).
